# AVSD-GER 技术栈地图

本文档按“技术方向 -> 具体技术 -> 系统作用”的方式盘点当前仓库的完整技术栈，覆盖模型 backbone、数据预处理、AVSD 前端、身份建模、跨模态对齐、LLM 纠错、闭环反馈、训练和评估。

---

## 1. 系统总览

AVSD-GER 是一套面向多说话人场景的 speaker-aware audio-visual speech recognition + generative error correction 系统。

整体链路：

```text
原始音频 / 原始视频
  -> 数据预处理
     -> 音频切片、16 kHz mono WAV
     -> 视频切片、mouth ROI、speaker mask
     -> session manifest / train JSONL
  -> AVSD 前端
     -> diarization
     -> active speaker detection
     -> face tracking
     -> mouth ROI extraction
  -> Backbones
     -> Whisper-large-v3 ASR
     -> AV-HuBERT Large VSR
     -> ECAPA-TDNN voice embedding
     -> InsightFace / ArcFace face embedding
     -> Llama-3-8B-Instruct GER
  -> C1 Cross-modal Identity Pool
  -> C2 ID-conditioned Audio-Visual Alignment
  -> LLM GER Head
  -> C3 Confidence-gated Closed-loop Feedback
  -> speaker-aware corrected transcript
```

核心定位：

```text
本系统不主张自己端到端解决 raw meeting video diarization。
它是一个 frontend-agnostic 后端框架：
给定不同质量的 diarization / ASD / face tracking / mouth ROI 前端，
通过身份条件对齐、生成式纠错和置信度闭环提升 speaker-aware transcript。
```

## 3. 数据预处理技术栈

### 3.1 音频预处理

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Audio slicing | ffmpeg | 从长会议音频中切出 per-turn WAV |
| Audio normalization | 16 kHz mono WAV | 匹配 Whisper、ECAPA 和训练输入 |
| Audio loading | soundfile | 读取 WAV 到 numpy / torch |
| Resampling | librosa | 非 16 kHz 音频重采样到 16 kHz |
| Enrollment quality selection | RMS dBFS | 选择更响、更清晰的说话人 enrollment 片段 |
| Audio quality gate | frame-energy SNR | DualGate 判断音频帧是否足够可靠 |
| Noise floor estimate | energy percentile | 用第 10 百分位 frame power 近似噪声底 |

音频预处理流程：

```text
meeting headset audio
  -> AMI segment XML
  -> start/end timestamps
  -> ffmpeg slice
  -> 16 kHz mono WAV
  -> per-turn audio file
  -> manifest.turns[].audio
```

Enrollment 音频构造：

```text
speaker headset channel
  -> first_seconds mode
     -> 直接取前 N 秒
  或
  -> turn_quality mode
     -> 过滤太短 / 太长 / 无文本 turn
     -> 计算 RMS
     -> 选择高质量 turn
     -> concat 成 enrollment WAV
```

相关脚本：

| 文件 | 作用 |
|---|---|
| `scripts/prepare_ami_manifest.py` | AMI XML + headset WAV -> session manifest |
| `scripts/prepare_lrs2_manifest.py` | LRS2 mp4 -> WAV + mouth ROI + manifest |
| `scripts/ami_visual_to_jsonl.py` | visual manifests -> train JSONL |

---

### 3.2 视频预处理

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Video slicing | ffmpeg | 从 meeting-length AVI / MP4 切出 turn-level 视频片段 |
| Frame decoding | OpenCV VideoCapture | 逐帧读取视频 |
| Mouth ROI extraction | AV-HuBERT style align_mouth | 生成 AV-HuBERT 所需嘴部输入 |
| Landmark detection | dlib frontal detector + CNN detector | 生产级人脸检测 |
| Facial landmark | dlib 68-point shape predictor | 找到嘴部 landmark 48-68 |
| Face alignment | mean-face affine transform | 对齐人脸，减少头部姿态影响 |
| Temporal smoothing | sliding window | 降低 landmark 抖动 |
| ROI crop | 96x96 grayscale | 匹配 AV-HuBERT VSR 输入 |
| Fallback ROI | OpenCV Haar cascade | 无 dlib 模型时的轻量替代 |
| ROI persistence | `.npy` | 保存 `[T, 1, 96, 96]` mouth ROI |

生产级 dlib mouth ROI：

```text
video frames
  -> dlib face detector / CNN detector
  -> 68-point landmarks
  -> interpolate missing landmarks
  -> smooth landmarks
  -> affine warp to mean face
  -> crop mouth region 96x96
  -> grayscale tensor [T, 1, 96, 96]
```

Haar fallback mouth ROI：

```text
video frames
  -> OpenCV Haar face detection
  -> estimate mouth center from lower face geometry
  -> interpolate missing centers
  -> temporal smoothing
  -> crop 96x96
  -> grayscale tensor [T, 1, 96, 96]
```

## 4. AVSD 前端技术栈

当前核心模型吃的是 turn-level manifest。raw meeting video 需要外部 AVSD frontend 先产生：

```text
who spoke when
active speaker face track
mouth ROI
speaker_mask_v
snr_per_tok / lip_conf_v
```

### 4.1 Frontend profiles

| Profile | 技术方向 | Diarization | Active Speaker | Face Tracking | Mouth ROI | 用途 |
|---|---|---|---|---|---|---|
| `oracle_turns` | 上限条件 | ground-truth segment boundaries + reference speaker labels | ground-truth / manually aligned active face track | dataset annotations / manually verified tracks | AV-HuBERT align_mouth.py 或 verified ROI | 排除前端误差，测 C1/C2/C3 上限 |
| `common_pyannote_lightasd` | 常用开源前端 | pyannote speaker-diarization-community-1 | Light-ASD 或 TalkNet | RetinaFace / InsightFace + SORT / ByteTrack | landmark crop / AV-HuBERT align_mouth.py | 主 practical baseline |
| `strong_sortformer_talknet` | 强前端参考 | NVIDIA NeMo Sortformer v2.1 或 pyannote Precision-2 | TalkNet 或 Light-ASD | RetinaFace / InsightFace + ByteTrack / DeepSORT | AV-HuBERT align_mouth.py | 验证强分割下 C1/C2/C3 是否仍有增益 |
| `degraded_pyannote` | 鲁棒性前端 | pyannote community + synthetic boundary jitter / drop / swap noise | Light-ASD / TalkNet + optional track noise | common frontend 后扰动 track assignment | same ROI extractor | 验证不完美前端下系统是否仍稳定 |

### 4.2 pyannote 方向

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Speaker diarization | pyannote speaker-diarization-community-1 | 产生说话人时间边界 |
| Strong diarization | pyannote Precision-2 | 强前端参考条件 |
| Degraded frontend | boundary jitter / speaker swap / turn drop | 压力测试 C1/C2/C3 鲁棒性 |
| Manifest input | turn start/end + speaker hypothesis | 给 SessionRunner 提供 turn list |

pyannote 在系统里的位置：

```text
raw meeting audio/video
  -> pyannote diarization
  -> turns: start, end, speaker label
  -> per-turn audio slicing
  -> AVSD-GER backend
```

注意：pyannote 不是当前 `pipeline.py` 内部模块，而是 upstream frontend 条件。

### 4.3 Light-ASD / TalkNet 方向

| 技术方向 | 具体技术 | 系统作用 |

## 6. C1: Cross-modal Identity Pool

### 6.1 MLP Identity Fuser

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Multimodal fusion | voice + face concat | 融合声纹和人脸身份 |
| Projection head | Linear projection | 对齐不同维度 embedding |
| Nonlinear fusion | MLP + GELU | 学习跨模态身份空间 |
| Metric space | L2 normalization | 支持 cosine retrieval |

结构：

```text
voice_emb: [192]
face_emb : [512]

voice_proj: Linear(192 -> 256)
face_proj : Linear(512 -> 256)

normalize voice_proj
normalize face_proj

concat: [512]
  -> Linear(512 -> 256)
  -> GELU
  -> Linear(256 -> 256)
  -> L2 normalize

z_id: [256]
```

### 6.2 IdentityPool

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Speaker memory | dict speaker store | 保存已注册 speaker |
| Retrieval | cosine similarity | 查询当前 utterance 最像谁 |
| Top-k identity | top_k = 3 | 返回候选 speaker |
| Unknown detection | min_av_consistency | 低相似度时判 unknown |
| Online update | EMA | 置信通过后刷新 voice / face embedding |

### 6.3 DualGate

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Audio quality gate | SNR >= tau_a | 过滤噪声音频 |
| Visual quality gate | lip_conf >= tau_v | 过滤遮挡 / 低置信嘴部帧 |
| Cross-modal quality control | audio AND visual | 只有 A/V 同时可靠才更新身份 |

默认阈值：

```text
tau_a_snr_db = 8.0
tau_v_lip_conf = 0.7
```

### 6.4 Agglomerative Cold Start

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Unsupervised clustering | AgglomerativeClustering | 无标签时发现 speaker cluster |
| Distance | cosine distance | 适合 L2-normalized embeddings |
| Auto-K | distance_threshold | 不固定 speaker 数 |
| Open-set | delta_unknown | 远离 cluster 的样本进入 unknown |

## 7. C2: ID-conditioned Audio-Visual Alignment

### 7.1 Token-level audio feature

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Frame-to-token pooling | mean pooling by word timestamp | 将 Whisper frame features 转成 word token features |
| Acoustic representation | `[N_tok, 1280]` | C2 query 输入 |
| Timestamp alignment | word start/end | 对齐 audio token 与 video frames |

### 7.2 Concat + Linear ID Injection

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Identity conditioning | concat audio token + z_id | 让 cross-attention 知道当前 speaker |
| Query construction | Linear projection | 生成 speaker-aware Q |
| Ablation contrast | FiLM 不作为默认 | 默认采用 spec 指定的 concat_linear |

公式：

```text
Q = Linear(concat(f_a_token, e_id))
K = Linear(f_v_segment)
V = Linear(f_v_segment)
```

### 7.3 Q-Norm + KV-Norm + Out-Norm

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Query normalization | Q-Norm | 稳定 audio+identity query 表征 |
| Key/Value normalization | KV-Norm | 稳定 visual feature 尺度 |
| Output normalization | Out-Norm | 稳定对齐输出 |
| Transformer stabilization | LayerNorm | 降低模态尺度不一致问题 |

数据流：

```text
asr_tok_feats [N, 1280] + z_id [256]
  -> ConcatLinearInject
  -> Q [N, 512]
  -> Q-Norm

vsr_feats [T, 1024]
  -> Linear
  -> KV [T, 512]
  -> KV-Norm

Q attends to KV
  -> residual
  -> FFN
  -> Out-Norm
  -> f_align [N, 512]
```

### 7.4 Soft-gated Cross Attention

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Multi-head attention | custom attention | audio token attend to visual frames |
| Speaker mask | key_padding_mask | mask 掉非当前 speaker 的视觉帧 |
| Soft reliability gate | additive log-bias | 低 SNR / 低 lip_conf 时降低注意力 |
| Gradient-friendly gating | log(gate + eps) | 不硬删，保留可学习路径 |

soft gate：

```text
gate[n, m] = min(SNR_token[n], lip_conf_frame[m])
attention_logits += log(gate + eps)
```

## 8. GER Head: LLM-based Generative Error Correction

### 8.1 Prompt-based GER

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| ASR post-correction | Llama GER | 纠正 Whisper 识别错误 |
| Speaker-aware correction | speaker special token | 保留 speaker identity |
| Audio hypothesis | ASR n-best / 1-best | LLM 主要依据 |
| Visual hypothesis | lip_hyp | 可选视觉文本辅助 |
| Multimodal context | `<AV_CTX>` | soft AV prefix 插入点 |

Prompt 结构：

```text
[Speaker: ID_i]
Audio hypothesis: ...
Visual hypothesis: ...
Aligned feature context: <AV_CTX>
Correct the transcript using the audio hypothesis as the main source.
Return only the corrected transcript text...
Output:
```

### 8.2 Q-Former Soft Prefix

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Soft prompt learning | learned queries | 将 variable-length f_align 压缩成固定长度 |
| Multimodal prefix | Q-Former style projector | 把 AV 对齐特征注入 LLM |
| LLM hidden projection | Linear -> 4096 | 匹配 Llama hidden size |

结构：

```text
f_align [N, 512]
learnable queries [16, 512]
  -> MultiheadAttention
  -> Linear(512 -> 4096)
  -> soft prefix [1, 16, 4096]
```

### 8.3 Speaker Special Token + Identity Bias

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Special token registration | tokenizer.add_special_tokens | 避免 `[Speaker: ID_i]` 被 BPE 拆碎 |
| Continuous identity injection | id_proj(z_id) | 将 256-d identity 注入 LLM embedding |
| Speaker-aware generation | token embedding bias | 让 GER 生成时感知当前 speaker |

流程：

```text
z_id [256]
  -> Linear(256 -> 4096)
  -> add to speaker token embedding
```

### 8.4 LoRA / Quantization

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Parameter-efficient tuning | LoRA r=16 alpha=32 | 只训练低秩 adapter |
| Attention adaptation | q/k/v/o proj | 调整注意力层 |
| MLP adaptation | gate/up/down proj | 调整 FFN 层 |
| Low-memory inference | 8-bit / 4-bit NF4 | 适配 16GB 以下 GPU |
| Auto precision | VRAM-based selection | 根据显存自动选择 fp16/int8/4bit |

LoRA target modules：

```text
q_proj, k_proj, v_proj, o_proj,
gate_proj, up_proj, down_proj
```

### 8.5 GER Safety Gates

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Artifact filtering | blacklist | 过滤 prompt echo / explanation |
| Length gate | max length ratio | 防止 LLM 输出过长 |
| Token overlap gate | ASR-GER overlap | 防止 LLM 偏离原音频 |
| Acoustic fallback | Whisper rescore comparison | GER 不如 ASR 时回退 1-best |
| Text cleaning | regex cleanup | 清理 label、引号、重复句 |

## 9. C3: Confidence-gated Closed-loop Feedback

### 9.1 Composite Confidence

| 信号 | 权重 | 技术方向 | 含义 |
|---|---:|---|---|
| Whisper acoustic rescore | 0.60 | acoustic verification | GER 文本是否符合音频 |
| AV consistency | 0.25 | identity verification | 当前 voice/face 是否匹配身份池 |
| N-best agreement | 0.10 | uncertainty estimation | ASR hypotheses 是否稳定 |
| LLM token confidence | 0.05 | generation confidence | LLM 输出是否自信 |

公式：

```text
total =
  0.60 * acoustic_rescore_conf
+ 0.25 * av_consistency
+ 0.10 * nbest_agreement
+ 0.05 * llm_conf
```

### 9.2 Closed-loop Policy

| 条件 | 动作 | 系统意义 |
|---|---|---|
| total < confidence_low | REIDENTIFY | 当前 speaker 可能错，跳过 top-1 重新查身份 |
| confidence_low <= total < confidence_mid | REALIGN | speaker 可能对，但对齐不够好，重新跑 C2 |
| total >= confidence_mid 且 acoustic_conf >= tau_update | ACCEPT_AND_UPDATE | 接受输出，并允许更新身份池 |
| total >= confidence_mid 且 acoustic_conf < tau_update | ACCEPT_NO_UPDATE | 接受输出，但禁止污染身份池 |

默认：

```text
confidence_low = 0.35
confidence_mid = 0.55
tau_update = 0.55
max_iters = 3
```

当前 AMI-safe 默认：

```text
enable_pool_update = false
```

即：默认可以做闭环判断，但不在线 EMA 更新身份池，避免评估时引入不受控漂移。

## 10. 训练技术栈

### 10.1 Stage 1: Identity-aware Alignment Pretraining

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Frozen backbones | Whisper / AV-HuBERT / ECAPA / ArcFace | 固定基础表征 |
| Cross-modal contrastive learning | Bidirectional InfoNCE | 学 voice-face 对齐 |
| Cold-start pseudo labels | Agglomerative clustering | 没有 speaker label 时生成 cluster |
| Quality filtering | DualGate | 只用高质量 A/V 样本 |
| Optimizer | AdamW | 训练 Identity Fuser |
| Warmup | linear warmup | 稳定初始训练 |

损失：

```text
L_stage1 = L_A->V + L_V->A
```

### 10.2 Stage 2: Multi-task Fine-tuning

| 技术方向 | 具体技术 | 系统作用 |
|---|---|---|
| Multi-task learning | CTC + GER CE + InfoNCE | 同时优化识别、纠错、身份 |
| Sequence alignment | char-level CTC | 让 f_align 可转写 |
| LLM supervised fine-tuning | teacher-forced CE | 训练 GER 输出 reference transcript |
| PEFT | LoRA adapter | 微调 Llama |
| Soft prefix training | Q-Former projector | 学习 AV context 注入 |
| Identity injection training | id_proj | 学习 speaker token identity bias |
| LR invariant | stage2_lr = stage1_lr * 0.1 | 保持二阶段稳定 |

损失：

```text
L_stage2 =
  1.0 * L_CTC
+ 1.0 * L_GER_CE
+ 0.5 * L_InfoNCE
```
